# MES 三层架构模块视觉交互原型设计

> **版本**: v1.0  
> **日期**: 2026-06-11  
> **状态**: 已确认，待实现  
> **范围**: 计划管理（Plan）、订单管理（Order）、工单管理（WorkOrder）三个前端页面的视觉交互原型

---

## 一、设计背景与目标

基于 MES 三层架构（Plan → Order → WorkOrder）的 API 文档、数据库设计和状态/权限设计文档，为三个模块设计统一的视觉交互原型。核心目标是：

- **状态不乱**：6 种状态的可操作按钮一目了然，禁用按钮明确不可操作
- **权责清晰**：不同角色的操作权限在 UI 上直接体现
- **行为可追溯**：级联操作前预览影响范围，操作后记录审计日志
- **贴合现场**：工单层支持员工现场数据采集、产量上报

## 二、整体布局方案

### 2.1 布局模式：主从布局（Master-Detail）

```
┌──────────────────┬────────────────────────────────────────┐
│   左侧列表 (380px)  │   右侧详情面板 (flex:1)                    │
│                  │                                        │
│  ┌搜索──────────┐ │  ┌标题 + 操作按钮──────────────────────┐ │
│  └──────────────┘ │  └────────────────────────────────────┘ │
│  ┌状态筛选────────┐ │  ┌状态流转步骤条───────────────────────┐ │
│  └──────────────┘ │  └────────────────────────────────────┘ │
│  ┌──────────────┐ │  ┌信息卡片 (grid: 2col)───────────────┐ │
│  │  列表项 1 ←  │ │  │ 基本信息  │  时间信息              │ │
│  │  列表项 2    │ │  └────────────────────────────────────┘ │
│  │  列表项 3    │ │  ┌关联数据 Tab ───────────────────────┐ │
│  │  列表项 ...  │ │  │ [关联订单/工单列表] [操作日志]      │ │
│  └──────────────┘ │  └────────────────────────────────────┘ │
│  ┌新建 + 分页─────┐ │                                        │
│  └──────────────┘ │                                        │
└──────────────────┴────────────────────────────────────────┘
```

三个模块各自独立页面，共享此布局模式。左侧展示主实体列表，右侧展示选中实体的详情和关联下层数据。

### 2.2 跨模块导航：Drawer 内联查看

- 从计划详情点击关联订单 → 右侧滑出订单 Drawer（不跳路由）
- Drawer 中展示订单基本信息 + 操作按钮 + 关联工单列表
- Drawer 底栏提供"打开完整页面"链接（可跳转到对应模块的独立页面）
- 从订单 Drawer 中的工单 → 同理展开工单 Drawer（支持三层穿透）

## 三、计划管理（Plan）交互设计

### 3.1 状态与操作按钮

| 状态 | 可用操作 | 可用编辑 | 可删除 | 可作废 |
|------|---------|---------|:---:|:---:|
| CREATED | PUBLISH | 全部字段 | ✅ | ❌ |
| RELEASED | CANCEL_PUBLISH, TERMINATE | planNum/时间/priority/remark | ❌ | ✅ |
| RUNNING | PAUSE | ❌ | ❌ | ❌ |
| PAUSED | RESUME | ❌ | ❌ | ❌ |
| COMPLETED | —（终态） | ❌ | ❌ | ❌ |
| TERMINATED | —（终态） | ❌ | ❌ | ❌ |

**按钮显示规则**：不可用的按钮以 `opacity:0.4` + `cursor:not-allowed` + `disabled` 呈现（而非隐藏），让用户知道该操作存在但当前不可用。

**状态提示条**：操作按钮下方显示提示信息（不同状态不同颜色背景），解释当前可做什么、不可做什么。

### 3.2 发布门禁检查面板

点击 PUBLISH 时，弹出"发布门禁检查"对话框，逐项展示三道门禁的校验结果：

1. **工艺存在性**：BOM → 工艺流程是否有效
2. **产线可用性**：是否有至少一条匹配的可用产线
3. **人员可执行性**：每道工序是否至少有一名具备技能的可用员工

每项校验显示 ✅ 通过 或 ❌ 失败（附修复建议）。三道全部通过后，"确认发布"按钮才可用。

### 3.3 级联操作预览弹窗

执行 PAUSE / RESUME / TERMINATE / CANCEL_PUBLISH 时，弹出"级联影响预览"弹窗：

- 显示影响范围表格：层级（本计划/关联订单/关联工单）、当前状态 → 目标状态、影响数量
- 附警告提示（如"暂停期间新工单不会自动开始"）
- 需用户点击"确认"后才执行

### 3.4 权限

所有状态操作（PUBLISH / CANCEL_PUBLISH / PAUSE / RESUME / TERMINATE）需"生产主管"角色。按钮旁不显示角色提示（由后端 403 拦截，前端通过获取用户角色信息预判按钮可见性）。

## 四、订单管理（Order）交互设计

### 4.1 操作按钮分区

订单操作按钮分为两个区域：

**人工可操作区**（左侧）：PAUSE / RESUME / TERMINATE — 正常按钮样式  
**系统驱动区**（右侧）：PUBLISH / CANCEL_PUBLISH / START_WORK / FINISH_WORK — 置灰 + 删除线 + "API已拦截"标签

两个区域之间用竖线分隔。系统驱动按钮仅作状态展示，不可点击。

### 4.2 状态与操作按钮

| 状态 | 人工可操作 | 系统驱动（展示） | 可编辑 |
|------|-----------|----------------|------|
| CREATED | — | PUBLISH | 全部 |
| RELEASED | TERMINATE | CANCEL_PUBLISH, START_WORK | quantity/时间/remark |
| RUNNING | PAUSE | FINISH_WORK | ❌ |
| PAUSED | RESUME | — | 仅 remark |
| COMPLETED | —（终态） | — | ❌ |
| TERMINATED | —（终态） | — | ❌ |

### 4.3 级联预览

与计划层逻辑一致，但影响范围仅到工单层（订单 → 工单）。

### 4.4 权限

人工操作（PAUSE / RESUME / TERMINATE）需"车间主任"或"生产主管"角色。

## 五、工单管理（WorkOrder）交互设计

### 5.1 双角色权限模型

| 操作 | 员工（派工人员） | 主管（车间主任/生产主管） |
|------|:---:|:---:|
| START_WORK | ✅ 需 userId 匹配 | ❌ |
| FINISH_WORK | ✅ 需 userId 匹配 | ❌ |
| PAUSE | ✅ 需 userId 匹配 | ✅ 可跨工单 |
| RESUME | ✅ 需 userId 匹配 | ✅ 可跨工单 |
| TERMINATE | ❌ | ✅ 仅 RELEASED/PAUSED |
| 上报产量 | ✅ 需 userId 匹配 | ❌ |

### 5.2 状态与操作按钮

| 状态 | 可用操作 | 可编辑 |
|------|---------|------|
| CREATED | PUBLISH（系统）、DELETE | 全部 |
| RELEASED | START_WORK（员工）、TERMINATE（主管） | plannedQuantity/时间/remark |
| RUNNING | PAUSE、FINISH_WORK（员工）、上报产量 | actualQuantity/scrapQuantity/remark |
| PAUSED | RESUME（员工/主管）、TERMINATE（主管） | 仅 remark |
| COMPLETED | —（终态） | ❌ |
| TERMINATED | —（终态） | ❌ |

### 5.3 产量上报弹窗（工单特有）

RUNNING 状态下员工点击"上报产量"按钮，弹出上报弹窗：

- **历史上报记录表格**：时间、本次产量、本次报废、累计产量
- **本次上报输入**：本次完成数量 + 本次报废数量（两个输入框）
- **上报后累计预览**：实时计算上报后的累计值
- 数据汇总后向上同步至订单的 quantityProduced / qualifiedProducts / defectiveProducts

### 5.4 关键工单（isCritical）标识

- 列表中关键工单显示 ⭐ 图标前缀
- hover ⭐ 时 Tooltip："关键工单 — 状态变更将影响订单状态聚合"
- 详情面板中显示"⭐ 关键工单 · 影响订单状态聚合"标签

## 六、表单编辑：状态感知的字段锁定

### 6.1 锁定方案

不可编辑字段以**只读文本 + 状态标签**呈现，而非置灰输入框：

```
工序:  焊接 (ID:10)  [🔒 已派工不可改]
```

优点：表单看起来干净、信息层级清晰、不会让用户产生"表单坏了"的疑惑。

### 6.2 各状态编辑规则

| 状态 | Plan | Order | WorkOrder |
|------|------|-------|-----------|
| CREATED | 全部可编辑 | 全部可编辑 | 全部可编辑 |
| RELEASED | planNum/时间/priority/remark | quantity/时间/remark | plannedQuantity/时间/remark |
| RUNNING | ❌ | ❌ | actualQuantity/scrapQuantity/remark |
| PAUSED | ❌ | remark | remark |
| COMPLETED | ❌ | ❌ | ❌ |
| TERMINATED | ❌ | ❌ | ❌ |

## 七、状态流转可视化

采用**横向步骤条**（类似 Element Plus Steps）：

- CREATED → RELEASED → RUNNING → COMPLETED 为主线
- PAUSED 和 TERMINATED 作为分支路径从主线下方向下引出
- **当前状态**：蓝色边框 + 呼吸动画 + "当前"标签
- **已完成状态**：绿色背景 + ✓ 图标
- **未到达状态**：灰色 + 透明度降低
- 每个节点 hover 显示 Tooltip（状态含义 + 触发条件）
- 点击非当前节点 → Toast "请通过状态操作按钮变更"

## 八、列表筛选设计

### 8.1 搜索与筛选

左侧列表顶部包含：

1. **搜索输入框** + 搜索按钮（支持按编号/名称搜索）
2. **状态快速筛选胶囊**：全部 / RUNNING / PAUSED / CREATED / 终态 — 显示各状态计数
3. **可展开的更多筛选**：优先级下拉框、产线下拉框（订单）、工序下拉框（工单）、人员下拉框（工单）

### 8.2 列表项信息展示

- 第一行：编号（含 ⭐ 如果是关键工单）+ 状态标签（右对齐）
- 第二行：名称/所属上层对象
- 第三行：关键指标（计划: 数量/完成量、订单: 产线/合格率、工单: 人员/产量进度）

### 8.3 分页

底部：新建按钮（左）+ 分页器（右）+ 总条数显示
